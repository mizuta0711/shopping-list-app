# ブラウザテスト チェックリスト

browser-tester エージェントが使用する共通チェックリスト。
画面固有のテスト項目は、各機能設計書の「4. ブラウザ評価計画」に記載する。

## 共通チェック項目

すべての画面で確認する項目:

### 表示・レンダリング
- [ ] ページが正常に表示される（白画面・エラー画面でない）
- [ ] コンソールにエラーが出ていない
- [ ] ローディング状態が適切に表示される（スケルトン/スピナー）

### レイアウト
- [ ] ヘッダー・ナビゲーションが表示され、機能する
- [ ] コンテンツが多い場合にスクロールが正しく機能する
- [ ] 要素のはみ出し・重なりがない

### スマホ表示（本アプリはスマホ専用）
- [ ] スマホ幅（375px）でレイアウトが崩れない
- [ ] 主要動線がタップで操作できる（タップ領域が十分）
- [ ] 縦スクロールが正しく機能する

### インタラクション
- [ ] ボタン・リンクのクリックが正しく動作する
- [ ] フォーム入力・送信が正常に動作する（該当する場合）
- [ ] エラー時に適切なフィードバックが表示される（該当する場合）

## 使い方

### スポットテスト時
1. 上記の共通チェック項目を対象画面に対して実施
2. 機能設計書の「4. ブラウザ評価計画」に固有テスト項目があれば、それも実施

### フルテスト時
1. 全画面に対して共通チェック項目を実施
2. 作業中の機能設計書があれば、その評価計画も実施
