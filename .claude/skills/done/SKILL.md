# 作業完了報告

作業が完了したときに呼び出すスキル。決められたフォーマットで報告を出力する。

## 手順

1. 直近の作業内容を把握する（TodoWrite のタスク一覧、直近のコミット、変更ファイル）
2. 以下のフォーマットで報告を出力する

## 報告フォーマット（必須）

```
### {作業名} 完了報告

| ステップ | 状況 |
|---------|------|
| 設計 | ✅ 具体的な内容 / ⬜ 不要（理由） |
| 設計レビュー | ✅ 具体的な内容 / ⬜ 不要（理由） |
| 実装 | ✅ 変更したファイル・内容の要約 |
| コードレビュー | ✅ 具体的な内容 / ⬜ 不要（理由） |
| ブラウザテスト | ✅ エビデンスパス / ⬜ 不要（UI変更なし等） |
| 設計書更新 | ✅ 具体的な内容 / ⬜ 不要（理由） |
| ビルド確認 | ✅ / ⬜ 不要（理由） |
| コミット | ✅ ハッシュとコミット概要 |
| プッシュ | ✅ / ⬜ ユーザー指示待ち |
```

## ルール

- 各ステップの「具体的な内容」は省略せず、何をしたか・どのファイルを変更したかがわかるように書く
- バグ修正など軽微な作業でも同じフォーマットで報告する
- 「⬜ 不要」の場合は理由を必ず記載する（例: S規模、設計書のみ、等）
- コミットハッシュは短縮形（7文字）で記載
- プッシュは CLAUDE.md のルールに従い、フェーズ完了時またはユーザー指示時のみ実施。勝手にプッシュしない
