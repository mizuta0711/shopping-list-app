# ブラウザテスト実施

## Step 1: テスト対象の特定

$ARGUMENTS が `full` の場合 → **フルテスト**として browser-tester を呼び出す。

$ARGUMENTS に画面URLが指定されている場合 → その画面を **スポットテスト**。

$ARGUMENTS が未指定の場合 → 直近の変更ファイルからUI変更を含む画面を特定:

```bash
git diff --name-only
```

変更がない場合は直近1コミット:

```bash
git diff --name-only HEAD~1
```

### 変更ファイル → 対象画面の対応（ヒューリスティック）

- `src/app/{path}/page.tsx` → 対応する URL パス
- `src/app/{path}/**/*.tsx` → そのルート配下の画面
- `src/features/{feature}/` → その feature が使われている画面
- `src/components/` → 影響範囲が広い可能性があるため、作業中の機能設計書を参照して対象画面を決定

UI関連の変更が見つからない場合は「UI変更なし。ブラウザテスト不要です。」と報告して終了。

## Step 2: 評価計画の確認

関連する機能設計書（`docs/features/` 配下の作業中の設計書）に「4. ブラウザ評価計画」が存在する場合、その内容に基づいてテストを実施する。

設計書がない場合は `.claude/browser-test-checklist.md` の共通チェック項目で実施する。

## Step 3: エビデンスフォルダの作成

browser-tester に委譲する**前に**、メインエージェント側でエビデンスフォルダを作成する:

```bash
# フォルダ名: YYYYMMDD_HHMMSS_テスト対象（例: 20260418_153000_example-page）
mkdir -p tests/browser-evidence/YYYYMMDD_HHMMSS_テスト対象/screenshots
```

## Step 4: browser-tester エージェントに委譲

エビデンスフォルダのパスを必ず含めて呼び出す:

```
# スポットテスト
Agent(subagent_type: "browser-tester", model: "sonnet", prompt:
  "スポットテスト。対象: {対象画面URL}。
   変更内容: {変更ファイルの概要}。
   確認観点: {評価計画の内容 or チェックリストの該当項目}。
   エビデンス保存先: tests/browser-evidence/{作成したフォルダ名}/")

# フルテスト
Agent(subagent_type: "browser-tester", model: "sonnet", prompt:
  "フルテスト。主要画面を網羅的に確認する。
   エビデンス保存先: tests/browser-evidence/{作成したフォルダ名}/")
```

## Step 5: レポート保存（メインエージェントが責任を持つ）

**サブエージェントが report.md を保存していない場合、メインエージェントが必ず自分で保存する。**

browser-tester の返答を受け取ったら:

1. `tests/browser-evidence/{フォルダ名}/report.md` が存在するか確認する
2. **存在しない場合** → browser-tester の返答内容をもとに、以下のフォーマットで `report.md` を自分で作成する
3. **存在する場合** → 内容に不足がないか確認し、不足があれば追記する


## Step 6: 結果報告

ユーザーに以下を報告する:
- 総合判定（✅ 合格 / ⚠️ 条件付き合格 / ❌ 不合格）
- テスト対象の画面一覧
- 発見した問題の件数と重要度
- エビデンスの保存先: `tests/browser-evidence/{フォルダ名}/report.md`
