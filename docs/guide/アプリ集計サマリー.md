## 1. SkillUp Mock（スキルコレクション）

### アプリのタイトル
**SkillUp Mock（スキルコレクション）** — AIペット型パーソナル学習アシスタント

### アプリの概要
キーワードを入力するだけで AI が学習コンテンツを生成し、ペットキャラクターと一緒に成長しながら技術スキルを身につけられる社内学習支援ツール。ライトモード（概要生成）とディープモード（ヒアリング→説明→質問→クイズ）の 2 フローで学習を進める。学習内容は自動でナレッジ化され、スキルは 5 段階でレベル管理される。

### ターゲット
SES 企業の客先常駐エンジニア（50〜300 名規模）。「調べるのが面倒」による学習放置の解消と、社内スキルの可視化がビジネス目標。

### 主要アーキテクチャ
- Next.js 16 (App Router) + React 19 + TypeScript 5 (strict)
- Tailwind CSS v4 + Zustand 5
- Prisma 6 + PostgreSQL
- NextAuth.js 4（Credentials Provider、role: user / admin）
- AI: Claude / Gemini / Azure OpenAI（切替可）
- Feature-based アーキテクチャ（`features/` 配下に機能別モジュール）
- レイヤー: Page → Hook → Service → Repository（Prisma）

### ファイルツリー（src/ 配下）

## 3. CodeReviewApp

### アプリのタイトル
**CodeReviewApp** — AI コードレビュー支援アプリ

### アプリの概要
GitHub リポジトリのコードを取り込み、Lint（ESLint 等）／Checkstyle（Java）／AI レビューを組み合わせて自動レビューを実施するアプリ。メインの Next.js アプリに加え、独立した 2 つのマイクロサービス（`checkstyle-api`: Google Cloud Run 上の Java Checkstyle 実行、`lint-api`: Supabase 連携の Lint 実行）を組み合わせる構成。

### ターゲット
コードレビュー工数を削減したい開発チーム／プロジェクトオーナー。GitHub 連携で既存リポジトリをそのままレビュー対象にできる。

### 主要アーキテクチャ
- メイン: Next.js 15 (App Router) + React + TypeScript + Tailwind CSS
- Prisma（PostgreSQL / SQL Server の 2 スキーマ対応）
- NextAuth.js（Credentials + signup）
- AI 診断: `aidiagnose` / `aireviewers` API
- GitHub 連携: Repo → Branch → Contents を取得する Proxy API
- サブサービス:
  - `checkstyle-api`: Node.js + Docker（Checkstyle 10.x JAR）、Cloud Run デプロイ
  - `lint-api`: Supabase + Lint 実行環境
- レイヤー: `repositories/` + `services/` + `stores/` + `hooks/` + `utils/validation/`

## 集計方法メモ

- ファイル数: TS/TSX（Next.js）または Kotlin（Android）の拡張子のみ
- 行数: 空行・コメントを含む（`wc -l`）
- class 数: TS は `^(export )?(abstract )?class `、Kotlin は `class \w+` 形式の宣言行
- interface 数: TS は `^(export )?interface `、Kotlin は `interface \w+`
- 関数・メソッド数（概算）:
  - TS: `function \w+(` 宣言 + `const \w+ = ... =>` 代入
  - Kotlin: `fun \w+(` 宣言
  - React のインライン JSX コールバック等は含まず、代入済みアロー関数は含む
- API エンドポイント数: `src/app/api/**/route.ts` 内で export された `GET/POST/PUT/DELETE/PATCH`（`export async function`、`export const = ...`、`export { handler as GET }` 形式すべて）
- DB テーブル数: `prisma/schema.prisma` 内の `^model \w+ {` 宣言数
- 開発期間・コミット数: `git log --reverse --format="%ad" --date=short | head -1`（初回）、`git log -1 --format="%ad" --date=short`（直近）、`git rev-list --count HEAD`（総数）。月別は `git log --format="%ad" --date=format:"%Y-%m" | sort | uniq -c`
