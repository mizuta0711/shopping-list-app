## 1. 概要

### 背景

[企画書 v0.3](../企画書.md) Phase 8。F-10（初回オンボーディング）を実装する。
データは端末ローカル保存のみであることを初回起動時にユーザーに明示する。

### 目的

- 初回起動時にウェルカム画面を全画面表示
- 「データはこの端末のみに保存される」ことを警告として表示
- 「はじめる」ボタンでメイン画面へ進む（`hasOnboarded: true` を保存）
- 2回目以降はメイン画面が直接表示される

### 方針

| 観点 | 判断 |
|------|------|
| 画面の種類 | モーダル overlay（独立ルートではなく、メイン画面に重ねて表示） |
| 表示判定 | `hydrated && !hasOnboarded` のとき表示 |
| 完了動作 | `setHasOnboarded(true)` で永続化、モーダルが消えてメイン画面が現れる |
| 表示場所 | `ShoppingMainView` 内部に配置（他の画面では表示しない。直接 URL でアクセスした特殊ケースは MVP では無視） |
| アクセシビリティ | role="dialog" + aria-labelledby + aria-modal="true" |

## 5. ブラウザ評価計画

### 5-3. 機能テスト項目

| # | 操作手順 | 期待結果 |
|---|---------|---------|
| 1 | LocalStorage を全削除 → アプリを開く | オンボーディング画面が表示される |
| 2 | 「はじめる」をタップ | オンボーディング画面が消え、メイン画面が表示される |
| 3 | リロードする | オンボーディング画面は表示されず、メイン画面が直接表示される |
