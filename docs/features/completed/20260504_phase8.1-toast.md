## 1. 概要

### 背景

ユーザーフィードバック: 追加・削除・購入・キャンセルなどの操作後、画面の視覚変化だけだと「確かに反映されたか」が分かりづらい場面がある。**トースト通知** を追加して操作の確認感を高める。

### 目的

- 主要な操作（追加・削除・購入・購入キャンセル・設定操作）後にトーストで結果を通知
- 既存の `window.alert()` をトーストに置換して UX を改善
- スマホ専用のため画面下部表示・短時間（1.5〜2秒）で消える

### 方針

| 観点 | 判断 |
|---|---|
| ライブラリ | **`sonner`**（React 19 互換、軽量、shadcn 等で標準採用、SSR セーフ） |
| 表示位置 | `bottom-center`（入力欄と被りすぎないよう、入力欄上に出るデフォルトを採用。リッチデバイス安全領域対応） |
| トーンと色 | デフォルト（中立）/ success（緑）/ error（赤） |
| 表示時間 | 1500ms（短く、買い物中の操作テンポを邪魔しない） |
| トースト発火位置 | コンポーネント側で `toast()` 呼び出し。ストアは UI 非依存を維持 |
| 既存 `alert` の扱い | `SettingsView` の `window.alert` は全てトーストに置換 |
| 確認ダイアログの扱い | `window.confirm`（全削除など破壊的操作）は**残す**（誤操作防止のため） |
| 重複防止 | 同一操作の連続発火時、`sonner` 既定の重複制御に任せる |

| 操作 | 文言 | トーン |

### 4-3. 実装方針

#### Toaster 設置（layout.tsx）

```tsx
import { Toaster } from "sonner";

<body>
  {children}
  <Toaster position="bottom-center" duration={1500} richColors closeButton={false} />
</body>
```

#### 一括追加の件数算出

`addItems` を呼ぶ前後で `useShoppingStore.getState().items.length` を取り、差分で新規追加件数を取得する。
重複スキップ件数 = 入力した有効件数（trim 後・空除外） − 実際に追加された件数。

```tsx
const before = useShoppingStore.getState().items.length;
addItems(names, scope);
const added = useShoppingStore.getState().items.length - before;
const trimmed = names.map(n => n.trim()).filter(Boolean).length;
const skipped = trimmed - added;
// 文言判定
```

#### 購入操作の toast 発火位置

`handleToggle` で `togglePurchased(id)` 呼び出し後、変化方向（PENDING↔PURCHASED）に応じて文言を出し分け。
**短時間に連続タップする場合は連発する**が、`sonner` の既定スタックで吸収される（過剰な制御は MVP 範囲外）。

## 7. 注意事項

- ストアの actions は UI 非依存を維持。トーストは呼び出し側の責務
- 連続タップ時のトースト連発は許容（MVP 段階では複雑な制御は不要）
- `window.confirm` はユーザーの誤操作防止のため残す（特に「全削除」）
