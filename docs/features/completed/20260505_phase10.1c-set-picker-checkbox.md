## 1. 概要

### 背景

Phase 10.1a でセット機能をリリース後、ユーザーから「セットから追加する際に**全商品が無条件で追加される**」のが不便との指摘。実運用では「セットの一部だけ要る」ケースが大半。

### 目的

セット選択後、**商品一覧をチェックボックスで表示し、ユーザーが選んだものだけリストに追加**できるようにする。

### 方針（対話で合意済み）

| 観点 | 判断 |
|---|---|
| 初期チェック状態 | **A-1 全件 ON**（既存挙動を踏襲し、不要なものだけ外す）。ただし C-1 により既存リスト内のアイテムは初期チェック OFF + 操作不可 |
| UI の出し方 | **B-1 同じボトムシート内で差し替え**（戻るボタンで第 1 段に戻る） |
| 既存リストにあるアイテム | **C-1 「✓ 既にリスト内」と表示してチェック不可**（disabled） |
| 空セット選択時 | **D トーストで「このセットは空です」と告知**し、第 1 段の一覧表示のまま |
| 全アイテムが既に存在するセット選択時 | 第 2 段に進入せずトースト「すべて既にリストにあります」で第 1 段維持（D の自然な拡張） |

## 3. 技術設計

### 3-1. 変更ファイル

| ファイル | 操作 | 内容 |
|---------|------|------|
| `src/features/shopping/components/SetPickerSheet.tsx` | 変更 | 2 段階化、selector で既存名を取得、チェック状態管理、フッターボタン |

**変更しないもの**:
- `useShoppingStore.addItems`（既存重複スキップロジックも併用するため変更不要・防御的）
- `useSetsStore`（型・アクションは現状のまま）
- `AddItemForm`（呼び出し側変更なし）
- 設計書 §5-5 の `before/after` 件数計算は不要に（事前に絞り込めるため `addItems` の重複スキップは「念のための保険」として残るが、ほぼゼロ件になる想定）

### 3-2. 内部 state 設計

```tsx
type Step = "list" | "items";

const [step, setStep] = useState<Step>("list");
const [selectedSet, setSelectedSet] = useState<ShoppingSet | null>(null);
const [checked, setChecked] = useState<Set<string>>(new Set());

const items = useShoppingStore((state) => state.items);

// アクティブ scope の PENDING アイテム名集合（C-1 の判定に使う）
const existingNames = useMemo(
  () =>
    new Set(
      items
        .filter((i) => i.scope === activeScope && i.status === "PENDING")
        .map((i) => i.name),
    ),
  [items, activeScope],
);
```

### 3-3. フロー

**第 1 段（list）でセットをタップしたとき**:

```ts
const handlePickSet = (set: ShoppingSet) => {
  if (set.items.length === 0) {
    toast(`「${set.name}」は空のセットです`);
    return;
  }
  // 既存にないアイテムだけ初期 ON（A-1 + C-1）
  const initial = new Set(
    set.items.filter((name) => !existingNames.has(name)),
  );
  if (initial.size === 0) {
    toast(`「${set.name}」のアイテムはすべて既にリストにあります`);
    return;
  }
  setSelectedSet(set);
  setChecked(initial);
  setStep("items");
};
```

**第 2 段（items）の戻る**:

```ts
const handleBack = () => {
  setStep("list");
  setSelectedSet(null);
  setChecked(new Set());
};
```

**第 2 段の確定**:

```ts
const handleConfirm = () => {
  if (!selectedSet) return;
  const names = selectedSet.items.filter((n) => checked.has(n));
  if (names.length === 0) return;
  const before = useShoppingStore.getState().items.length;
  useShoppingStore.getState().addItems(names, activeScope);
  const added = useShoppingStore.getState().items.length - before;
  toast.success(`「${selectedSet.name}」から ${added}件 追加しました`);
  handleClose(); // 全シートを閉じる
};
```

**チェックトグル**:

```ts
const handleToggle = (name: string) => {
  if (existingNames.has(name)) return; // C-1: 既存はトグル不可
  setChecked((prev) => {
    const next = new Set(prev);
    if (next.has(name)) next.delete(name);
    else next.add(name);
    return next;
  });
};
```

### 3-4. 第 2 段の画面レイアウト（375px）

```
┌──────────────────────────────────┐
│ ← カレーセット                ✕  │ ← ヘッダー（戻る + タイトル + 閉じる）
├──────────────────────────────────┤
│ ☑ 玉ねぎ                          │ ← チェック ON
│ ☑ にんじん                        │
│ ☑ じゃがいも                      │
│ ☐ 豚肉                            │ ← OFF（任意で外せる）
│ ✓ カレールー    既にリスト内      │ ← C-1: disabled, 灰色
├──────────────────────────────────┤
│         [追加 (3 件)]             │ ← フッター（sticky）
└──────────────────────────────────┘
```

- ヘッダー左: `←` 戻る（第 1 段へ） / 右: `✕` 全閉じる
- 既存リスト内アイテムは行全体を `text-gray-400` + `cursor-not-allowed`、チェックの代わりに `Check` アイコン + 「既にリスト内」ラベル
- フッターはシート下部に `sticky` 固定、件数 0 のとき disabled、`bg-gray-900 text-white` で目立たせる

### 3-5. アクセシビリティ

- 第 2 段ヘッダーの `aria-labelledby` は `set-picker-title`（既存）から `set-picker-items-title` に切り替え
- 戻るボタン: `aria-label="セット一覧に戻る"`
- チェック行: `<button role="checkbox" aria-checked={checked.has(name)} aria-disabled={existingNames.has(name)}>`
- フッター: `aria-label="選択した X 件を追加"`、disabled 時は `aria-disabled="true"`
- `Esc` キー: 第 2 段なら `handleBack`、第 1 段なら `handleClose`
- backdrop タップ: 全閉じる（第 1 段と同じ挙動を維持）。第 2 段から第 1 段に戻りたいときは戻るボタンを使う

### 3-6. トースト文言

| シナリオ | 文言 |
|----------|------|
| 空セット選択時 | 「{セット名} は空のセットです」 |
| 全件既存時 | 「{セット名} のアイテムはすべて既にリストにあります」 |
| 追加成功 | 「{セット名} から N 件 追加しました」 |

旧文言「{セット名} を追加しました（N 件 / M 件は重複）」は **使わなくなる**（事前に絞り込むため）。

## 6. 注意事項

- 第 2 段に進む前に「空セット」「全件既存」を 2 種のガードで弾くことで、ユーザーが何もできない第 2 段に到達する不愉快を回避する
- `existingNames` は `activeScope` に依存するため、scope が変わると判定がリアクティブに切り替わる（メモ化済み）
- チェック状態は **第 2 段に入るたび初期化**する。セット間で状態を持ち越さない
- `addItems` の既存重複スキップは保険として残す（同期ラグ等で `existingNames` と DB 状態がずれた場合の防御）
