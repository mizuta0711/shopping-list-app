## 1. ストアフック (Zustand)

| フック名 | ファイルパス | 説明 |
|---------|-------------|------|
| `useShoppingStore` | `src/features/shopping/stores/shoppingStore.ts` | 買い物リストの状態管理。LocalStorage persist。Phase 9 で `setItems` / `applyServerChanges` 追加。Phase 11 で `updateItemName` / `restoreItem` 追加。Phase 13 で `restoreItem`（アンドゥ削除用）削除（confirm 方式へ巻き戻し）。Phase 10.2 で `addItem` / `addItems` が `listId` 必須化、`relocateItems` / `applyListDeleted` / `remapListIds` 追加 + LocalStorage v2→v3 マイグレーション (`migrateToV3` + バックアップ + isV2 型ガード) |
| `useSyncStore` | `src/features/sync/stores/syncStore.ts` | 同期状態管理 (status, lastUpdatedAt, pending Set, clockSkewMs)。Phase 10.1b で sets 用フィールド追加: `lastSetsUpdatedAt` / `pendingSetUpsertIds` / `pendingSetDeleteIds` / `markSetUpsert` / `markSetDelete` / `consumeSetPending` / `setLastSetsUpdatedAt`。Phase 10.2 で lists 用フィールド追加: `lastListsUpdatedAt` / `pendingListUpsertIds` / `pendingListDeleteIds` / `markListUpsert` / `markListDelete` / `consumeListPending` / `setLastListsUpdatedAt`。Set 型は partialize で除外 |
| `useSetsStore` | `src/features/shopping/stores/setsStore.ts` | 買い物セット（カレーセット等）の状態管理。LocalStorage 別キー (`shopping-list-app:sets:state`) で persist。Phase 10.1a で追加。アクション: `addSet` / `updateSet` / `deleteSet` / `reset`。Phase 10.1b で `setSets` / `applyServerChanges` 追加（クラウド同期化）。Phase 10.4 で `ShoppingSet.listId` 必須化 + `applyListDeleted(deletedListId, unclassifiedId)` / `remapListIds(remappedIds: Record<string, string>)`（※ `useShoppingStore.remapListIds` の `(fromIds, toId)` シグネチャとは異なり sets 用は Record 形式）/ `repairMissingListIds(unclassifiedId)` アクション追加。`addSet` が `listId` 必須パラメータを受け取るよう変更。LocalStorage v1 → v2 マイグレーション（全セットに `listId = 未分類 ID` を付与 / バックアップ + 中断時冪等再試行）。persist version v2 に更新 |
| `useListsStore` | `src/features/shopping/stores/listsStore.ts` | 買い物リスト（複数リスト機能）の状態管理。Phase 10.2 で追加。アクション: `addList` / `updateList` / `deleteList` / `setLists` / `applyServerChanges` / `ensureUnclassified` / `reset`。`addList` / `updateList` / `deleteList` は `useSyncStore.markListUpsert` / `markListDelete` を直接呼ぶ。「未分類」(system: true) はユーザー作成リストとは別枠で常に存在、削除・改名・絵文字変更不可 |
| `useActiveListStore` | `src/features/shopping/stores/activeListStore.ts` | アクティブリスト ID + 移動モード state。Phase 10.2 で追加。アクション: `setActiveListId` / `enterMoveMode` / `exitMoveMode` / `toggleMoveSelection` / `clearMoveSelection` / `reset`。`activeListId` のみ永続化 (`shopping-list-app:active-list`)、`moveMode` (enabled / selectedItemIds) はメモリのみ。`setActiveListId` 内部でリスト切替時に moveMode を OFF にリセット |

## 2. 同期フック (Phase 9)

| フック名 | ファイルパス | 呼び出すAPI / 連携先 | 説明 | 実装Phase |
|---------|-------------|---------------------|------|----------|
| `useLocalStorage` | `src/features/sync/hooks/useLocalStorage.ts` | (LocalStorage) | SSR 対応の汎用 hook。null キー時 no-op。useInitialMerge でユーザー単位フラグ管理に使用 | Phase 9 |
| `useSyncStatus` | `src/features/sync/hooks/useSyncStatus.ts` | useSyncStore | UI コンポーネント (SyncStatusDot 等) で同期状態を購読 | Phase 9 |
| `useSyncOnMount` | `src/features/sync/hooks/useSyncOnMount.ts` | `useSession` + syncOrchestrator | session 状態に応じて orchestrator を起動/停止し、初回 pull を実行 | Phase 9 |
| `useInitialMerge` | `src/features/sync/hooks/useInitialMerge.ts` | `orchestrator.mergeLists` + `merge` (items) + `mergeSets` | hasMerged フラグで分岐: マージ済 → `pullListsOnce` / `pullOnce` / `pullSetsOnce` を直列実行 / 未マージ → **lists 先行直列マージ → items + sets を `Promise.all` で並列マージ**の 2 フェーズ。lists merge の `remappedUnclassifiedIds` を items の listId リマップに使うため lists が必ず先。Phase 10.4 で sets merge 前にも `useSetsStore.remapListIds(remappedUnclassifiedIds, unclassifiedId)` を適用して sets の listId を正規化してから merge 送信する（C3 対応）。`uploadedCount` / `downloadedCount` は 3 種を合算してトースト | Phase 9 |
| `useSyncOrchestrator` | `src/components/providers/SyncProvider.tsx` | (Context 取得) | SyncProvider が配信する orchestrator インスタンスを取得。タスク #10 で SyncStatusSheet が利用 | Phase 9 |

---

## 3. フック設計パターン

- **ストアフック**: Zustand `create + persist`。Set 型などシリアライズ不可な値は `partialize` で除外
- **API 連携フック**: 直接 fetch せず、サービス層 (`syncClient`) を経由する。リトライ・AbortController はサービス層に集約
- **副作用フック**: `useEffect` cleanup で必ずリスナー解除。HMR 対応のため orchestrator は factory 関数化 (`createSyncOrchestrator()`) し、ref で保持
