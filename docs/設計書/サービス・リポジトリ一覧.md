# サービス・リポジトリ一覧

## 概要

MVP は LocalStorage のみで Service 層は未使用だった。Phase 9 でクラウド同期 + 認証を導入し、API route から直接 Prisma Client を呼ぶ薄い層 + ヘルパーユーティリティ群を追加した。本格的な Service 層 / Repository 層は将来フェーズで分離する想定。

---

## 1. 認証関連 (`src/lib/auth.ts`)

| 関数 / 定数 | 説明 |
|-----------|------|
| `handlers` | Auth.js v5 の `{ GET, POST }` ハンドラー。`src/app/api/auth/[...nextauth]/route.ts` から re-export される |
| `auth()` | サーバー側 (route handler / Server Component) で現在のセッションを取得 |
| `signIn(provider)` | プログラム的なサインイン |
| `signOut()` | プログラム的なサインアウト |

設定: `PrismaAdapter(prisma)` + `session.strategy: "database"` + Google provider + `pages.signIn: "/login"`。Session callback で `user.id` を `session.user.id` に注入。

## 3. データアクセス

Phase 9 / 10.1b では Repository 層を作らず、API route から直接 `prisma` (`src/lib/db.ts`) を呼ぶ。route 内で `prisma.$transaction` を使い LWW 判定 + tombstone upsert を1トランザクションで実行する。

Phase 10.2 で「未分類」リスト保証ロジックの複雑化を理由に **`src/lib/services/listSyncService.ts`** を新設（最初の Service 層）。items / sets の Service 層化方針は `docs/設計書/技術負債と将来課題.md` MAINT-3 で横断課題として登録済み。Phase 10.4 で `ensureUnclassifiedList` を sets API (`PUT /api/sync/sets` / `POST /api/sync/sets/merge`) からも呼び出す形に拡張した（sets 専用 Service は作らず route 層から直接流用）。

### 3-1. Service 層 (`src/lib/services/`)

| ファイル | エクスポート | 説明 | 実装Phase |
|---------|------------|------|----------|
| `listSyncService.ts` | `ensureUnclassifiedList(tx, userId): Promise<ShoppingList>` | 各ユーザーごとに「未分類」リストを 1 件保証。`partial unique index WHERE system = TRUE` + `INSERT ... ON CONFLICT DO NOTHING` で同時呼び出しでも常に 1 件を返す。Prisma transaction client 内専用。Phase 10.2 は lists API のみで使用。Phase 10.4 から sets API (`PUT /api/sync/sets` / `POST /api/sync/sets/merge`) でも `listId` 補完のために利用 | Phase 10.2 |

## 4. クライアント側同期サービス (`src/features/sync/services/`)

| ファイル | エクスポート | 説明 | 実装Phase |
|---------|------------|------|----------|
| `reconcile.ts` | `reconcile({ local, serverChanges, serverDeletes, rejected })` | 純粋関数。アイテム用 LWW 判定 + 上書き発生件数を返す | Phase 9 |
| `reconcile.ts` | `reconcileSets({ local, serverChanges, serverDeletes, rejected })` | 純粋関数。セット用 LWW 判定 + 上書き発生件数を返す | Phase 10.1b |
| `syncClient.ts` | `syncClient.pull / push / mergeOnLogin`, `HttpError` | fetch ラッパー。指数バックオフ (1s/2s/4s) + AbortController + 401 handling | Phase 9 |
| `syncClient.ts` | `syncClient.pullSets / pushSets / mergeSetsOnLogin` | セット用 fetch ラッパー。既存 pull/push/mergeOnLogin と同じエラーハンドリング | Phase 10.1b |
| `syncOrchestrator.ts` | `createSyncOrchestrator()` (factory) | `start/stop/pullOnce/pushPendingNow/retry/merge`。`useShoppingStore.subscribe` + debounce 1.5s + window online/offline/focus イベント。HMR・SSR 対応。merge は初回ログイン時のサーバーマージで使用 | Phase 9 |
| `syncOrchestrator.ts` | `pullSetsOnce / pushPendingSetsNow / mergeSets` | factory 戻り値に追加。`useSetsStore.subscribe` + debounce 1.5s で sets の差分を検知し push。mergeSets は初回ログイン時のセットマージで使用 | Phase 10.1b |
| `reconcile.ts` | `reconcileLists({ local, serverChanges, serverDeletes, rejected })` | 純粋関数。リスト用 LWW 判定 + 上書き発生件数 (SYSTEM_PROTECTED 含む) を返す | Phase 10.2 |
| `syncClient.ts` | `syncClient.pullLists / pushLists / mergeListsOnLogin` | リスト用 fetch ラッパー。同パターン | Phase 10.2 |
| `syncOrchestrator.ts` | `pullListsOnce / pushPendingListsNow / mergeLists` | factory 戻り値に追加。listsStore のアクションが mark を直接呼ぶ + debounce 1.5s。push 順序は lists → items → sets の直列。pull 後に削除済みリストの listId を未分類へ連鎖更新 + activeListId を未分類へフォールバック (`ensureActiveListIsValid`) | Phase 10.2 |

## 4. サービス間の依存関係

```
src/app/api/sync/items/route.ts
  ├─ requireSession (sync-helpers)
  ├─ SyncPullQuerySchema / SyncPushSchema (sync-schemas)
  ├─ toDTO (dto)
  └─ prisma (lib/db)

src/app/api/sync/merge/route.ts
  ├─ requireSession (sync-helpers)
  ├─ SyncMergeSchema (sync-schemas)
  ├─ toDTO (dto)
  └─ prisma (lib/db)

src/app/api/sync/sets/route.ts
  ├─ requireSession (sync-helpers)
  ├─ SetsSyncPullQuerySchema / SetsSyncPushSchema (sync-schemas)
  ├─ setToDTO (dto)
  ├─ ensureUnclassifiedList (services/listSyncService)   ← Phase 10.4 追加
  └─ prisma (lib/db)

src/app/api/sync/sets/merge/route.ts
  ├─ requireSession (sync-helpers)
  ├─ SetsSyncMergeSchema (sync-schemas)
  ├─ setToDTO (dto)
  ├─ ensureUnclassifiedList (services/listSyncService)   ← Phase 10.4 追加
  └─ prisma (lib/db)

src/app/api/sync/lists/route.ts
  ├─ requireSession (sync-helpers)
  ├─ ListsSyncPullQuerySchema / ListsSyncPushSchema (sync-schemas)
  ├─ listToDTO (dto)
  ├─ ensureUnclassifiedList (services/listSyncService)
  └─ prisma (lib/db)

src/app/api/sync/lists/merge/route.ts
  ├─ requireSession (sync-helpers)
  ├─ ListsSyncMergeSchema (sync-schemas)
  ├─ listToDTO (dto)
  └─ prisma (lib/db)

src/app/api/auth/[...nextauth]/route.ts
  └─ handlers (lib/auth)
```
