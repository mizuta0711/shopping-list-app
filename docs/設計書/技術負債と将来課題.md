# 技術負債と将来課題

複数 Phase 横断の改善候補。各 Phase の機能設計書 (`docs/features/`) には書ききれない、長期的な技術負債や横断的な改善案を集約する。

## 運用ルール

- **追加タイミング**: コードレビュー・設計レビュー・ブラウザテストで「次フェーズ検討」「許容（将来対応）」とした項目を、各 Phase 完了時にここへ転記する
- **着手判断**: 新しい Phase の計画時に本ドキュメントを参照し、優先度の高いものを次 Phase のスコープに取り込む。優先度が同等なら関連性の高い Phase に巻き取る
- **削除タイミング**: 解消したら該当行に「✅ 解消 (Phase X.Y で対応、コミット xxxxxxx)」と記載。半年経過した解消済み項目は本ドキュメント末尾の「アーカイブ」セクションへ移動
- **Phase 固有の改善案**: 該当 Phase の機能設計書 §7「将来タスク」に書く。本ドキュメントは複数 Phase 横断のもののみ
